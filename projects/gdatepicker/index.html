Installation

First of all download the build code from: GitHub


jQuery

Make sure you've included a jQuery & UI package with $.ui.position and $.widget modules, I've shown them being loaded remotely, you can download and include the libraries yourself if you wish.
jQuery | UI

<script src="http://code.jquery.com/jquery-1.10.2.min.js"></script>
<script src="http://code.jquery.com/ui/1.10.3/jquery-ui.min.js"></script>

Dependencies

Then include the datepicker into your HTML along with it's dependencies. Scroll wheel plugin is optional, but it greatly adds to the UX.
Date.js | jQuery Mousewheel

<script src="js/date.js"></script>
<script src="js/jquery.mousewheel.js"></script>

<script src="js/jquery.ui.gdatepicker.js"></script>

Stylesheets

Make sure you've got the stylesheets loading for the datepicker, and if you plan to use a theme that the theme css is loaded, too.

<link rel="stylesheet" type="text/css" href="css/jquery.ui.gdatepicker.css">

Initialize

All that's left now is to initialize the plugin.

At it's most basic you can just run the datepicker on an <input> element:

$(document).ready(function() {
  $('input.element').gdatepicker();
});

This will give you a new <input> element which has the datepicker bound to it, and your other <input> will be hidden and slaved, meaning you can still use it for sending data back to the server.


Options

Below are the datepicker's default options which can be overidden by supplying an object to the datpicker call.

// The Datepicker's Default Options:
$('input.element').gdatepicker({

  placeholder:  "Pick a date...",						
  // string:	
  // eg: "Pick me!"
  // generated input's placeholder if original input doesn't have.
								
  selected: [],										
  // string, array
  // eg: "", [ 31, 12, 2012 ]
  // default date that is selected/highlighted. 
  // leave as blank string for none
  // overridden if input has "value=" with same format as "format" option
  							
  format:  "dd-MM-yyyy",							
  // string 
  // eg: "dd-MM-yyyy"
  // format of original input

  formatOutput:  "MMM dSX, yyyy",						
  // string
  // eg: "dd of MMMM, yyyy" 
  // format of generated output

  days:  ['M','T','W','T','F','S','S'],			
  // array
  // eg: ['L','M','M','J','V','S','D']
  // days of week in header

  position:  [3,0],									
  // array
  // eg: [0,0] 
  // floated position of calendar in relation to input

  scrollSpeed:  300,									
  // integer
  // eg: 300
  // how fast the calendar scrolls up and down

  overlayType:  "both",									
  // string, bool
  // eg: false, "both", "month", "year"
  // do we show the overlay for scrolling months/years

  overlayAnimation:  "drop",									
  // string
  // eg: "drop", "fade"
  // how the overlay is animated in and out

  theme:  false									
  // string, bool
  // eg: false, "dark", "mint", "..."
  // sets the css theme style, supply your own string for custom

});

Events

Here are hookable methods, you can add your own code to fire when these events are triggered.

// The Datepicker's Hookable Events:
$('input.element').gdatepicker({

  show: function(e) {},
  hide: function(e) {},
  // events fired when calendar hides or shows
	
  selectDay: function(e,ui) {},
  // event fired when user selects a date
	
  changeValue: function(e,ui) {},
  // event fired when value of input changes
  // (usually from keyboard entry)
	
  invalidDate: function(e,ui) {},
  // event fired when a invalid date is entered
  // (usually from keyboard entry)
	
  changeMonth: function(e,ui) {},
  changeYear: function(e,ui) {},
  // event fired when user changes month or year
	
  clearInput: function(e,ui) {}
  // event fired when used clicks "x" to clear input
	
});
